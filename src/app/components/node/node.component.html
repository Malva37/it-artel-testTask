1. Що таке файл package.json?
    2. package.json - це файл, що містить інформацію про Node.js проект та його залежності. Він містить різноманітні метадані про проект, такі як: назва, версія, автор, ліцензія, скрипти, залежності та багато іншого. Цей файл також дозволяє здійснювати встановлення та керування залежностями проекту. Коли виконується команда npm install в директорії проекту, використовуючи package.json, npm автоматично завантажує та встановлює всі залежності проекту. також там иожна прописати скрипти, які можна виконувати з командного рядка, наприклад, запуск тестів або збирання проекту.
  2. Які є типи запитів HTTP?
    • GET-запит передає дані через URL-адресу та використовується для отримання HTML-сторінок, зображень, іншого контенту.
    • POST-запит передає дані у тілі запиту та зазвичай використовується для створення нових ресурсів, оновлення існуючих даних або відправки форм.
    • PUT-запит передає дані у тілі запиту та зазвичай використовується для оновлення даних.
    • DELETE-запит передає дані у тілі запиту та використовується для видалення даних.
    • PATCH-запит передає дані у тілі запиту та зазвичай використовується для оновлення частини даних в ресурсі.
  3. Поясніть різницю між локальним і глобальним встановленням пакетів npm
    4. Глобальне встановлення пакетів npm означає, що пакет буде доступний для всіх проектів на комп'ютері. Локальне встановлення пакетів npm означає, що пакет буде доступний лише для проекту, в якому він встановлений.
  4. Поясніть порядок виконання прослуховувачів подій у Node.js. Що буде результатом цього коду і чому? Розглянемо цей код: 
          
    import { EventEmitter } from 'events';
    const eventEmitter = new EventEmitter();

    eventEmitter.on('myEvent', (data) =>  {
        console.log(data, '- FIRST');
    });

    console.log('Statement A');

    eventEmitter.on("myEvent", data => {
        console.log(data, '- SECOND');
    });

    eventEmitter.emit('myEvent', 'Emitted Statement');

    console.log("Statement B");
    5. Консоль буде виглядати наступним чином: 
          
        Statement A
        Emitted Statement - FIRST
        Emitted Statement - SECOND
        Statement B
    Вивід "Statement A№ відбудеться першим, адже цей рядок є синхронним і знаходиться перед реєстрацією подій. Згодом відбудеться вивід "Emitted Statement - FIRST Emitted Statement - SECOND", тому що всі прослуховувачі події myEvent виконуються послідовно зареєстрованим порядком. Після цього буде виведений рядок "Statement B", оскільки цей рядок є також синхронним і знаходиться після виклику eventEmitter.emit().
  5. Поясніть, що не так із використанням async/await у циклі forEach. Чи працює він належним чином? Ви можете це виправити? Розглянемо цей код: 
          
    import fs from 'fs-promise'

    async function printFiles ()
    const files = await getFilePaths() // Assume this works fine

    files.forEach(async (file) => {
        const contents = await fs.readFile(file, 'utf8')
        console.log(contents)
    })
    }

    printFiles()
    6. Головна проблема в тому, що цикл forEach не очікує на Promise. Потрібно переписати код в середина функції 'printFiles' на наступний: 
          
        for (const file of files) {
          const contents = await fs.readFile(file, 'utf8')
          console.log(contents)
        }
  6. Що саме робить module.exports у Node.js і яким би був простий приклад?
    7. 'module.exports' є об'єктом в Node.js, який використовується для встановлення зв'язку між файлами модулів Node.js. Ось приклад використання:

    у файлі, що експортує: 

        
      function add(a, b)  {
        return a + b;
      }
      module.exports = add;

    у файлі, що імпортує: 

        
      const add = require('./add');
      const sum = add(2, 3);
      console.log(sum); // 5
